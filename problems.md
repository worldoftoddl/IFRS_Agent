# 진단된 문제점 및 개선 과제

실제 실행 로그(`run-019d2331-...json`) 분석을 통해 발견된 문제점.

---

## 1. Step 1(기준서 식별)에서 top-1만 선택하는 구조적 한계

**현상**: "개념체계 중 측정에 관한..." 질문에 K-IFRS 1113(공정가치 측정)이 top-1으로 선택됨.
개념체계는 유사도 0.347로 2~3위에 머무름.

**원인**: `_step1_identify_standard`가 top-3을 반환하지만, `search_ifrs`에서 `standards[0][0]`(top-1)만 사용.
개념체계처럼 광범위한 주제를 다루는 기준서는 특정 키워드 유사도가 낮아 상위에 오르기 어려움.

**개선 방안**:
- Step 2에서 top-1 기준서만 검색하지 않고, **상위 복수 기준서에서 함께 검색** (예: top-3 기준서의 chunks를 통합 검색)
- 유사도 차이가 작은 경우(예: top-1과 top-2 차이 < 0.05) 복수 기준서 검색 자동 적용
- `search_ifrs`에 `standard_id` 선택 파라미터를 추가하되, 개념체계만을 위한 하드코딩이 아닌 범용적 설계

---

## 2. 개념체계 본문 문단이 벡터 검색에서 잘 나오지 않음

**현상**: 개념체계가 선택되더라도 문단 6.5(역사적 원가), 6.21(현행원가), 6.26(공정가치), 6.61(이행가치) 등 핵심 문단이 검색 결과에 포함되지 않음.
용어 정의 테이블만 반환됨.

**원인 후보** (_IFRS_parsing 프로젝트에서 확인 필요):
- 개념체계의 chunks authority 값이 올바르게 태깅되었는지 (authority=1 필터에 걸리는지)
- 개념체계의 청크 분할이 적절한지 (문단 단위로 제대로 잘렸는지)
- 개념체계 문단의 임베딩 품질 (요약 대비 본문 임베딩 유사도가 너무 낮은지)

**확인 방법**:
```sql
-- 개념체계 chunks의 authority 분포 확인
SELECT authority, count(*) FROM chunks
WHERE standard_id = '재무보고 개념체계'
GROUP BY authority ORDER BY authority;

-- 측정 관련 문단이 존재하는지 확인
SELECT para_number, section_title, authority, char_count
FROM chunks
WHERE standard_id = '재무보고 개념체계'
  AND content_text LIKE '%이행가치%'
ORDER BY para_number;
```

## 3. Agent 반복 검색으로 인한 토큰 낭비

**현상**: 동일 질문에 대해 `search_ifrs`를 8회 반복 호출, 59K 토큰 소비.
매 검색마다 유사한 쿼리를 변형하여 재시도하지만, 근본적으로 같은 결과 반환.

**원인**: 시스템 프롬프트에 "반드시 도구를 사용하세요" + "확실하지 않으면 추측하지 마세요"만 있고,
반복 검색 제한이나 검색 실패 시 행동 지침이 없음.

**개선 방안**:
- 시스템 프롬프트에 반복 호출 제한 규칙 추가 (예: "동일 도구를 동일 목적으로 3회 이상 호출하지 마세요")
- 검색 실패 시 행동 지침: "있는 정보를 바탕으로 답변하고, 추가 확인이 필요하다고 안내하세요"

---

## 4. Step 1 유사도 임계값 없음

**현상**: 유사도가 매우 낮은 기준서(예: 0.24)도 top-1이면 선택되어 무관한 검색 결과 반환.

**개선 방안**:
- 최소 유사도 임계값 설정 (예: 0.3 미만이면 "관련 기준서를 찾을 수 없습니다" 반환)
- 또는 Agent에게 유사도 점수를 보여주고 판단하게 하기
