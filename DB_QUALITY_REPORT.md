## 1. 문제 요약

K-IFRS 벡터 DB의 **IAS 계열(제10XX호) 기준서 9건**에서 본문(main, authority=1) 문단이
적용범위(1~7번대)만 적재되어 있고, **핵심 내용(정의/인식/측정/표시/공시) 문단이 전부 누락**되어 있다.

검색 파이프라인(하이브리드 RRF, authority 동적 필터)은 정상 동작하지만,
**DB에 데이터가 없으므로 어떤 검색 방법으로도 해당 문단을 찾을 수 없다.**

## 3. 근거 데이터

### K-IFRS 1021 (환율변동효과) — 대표 사례

```
DB에 존재하는 main(authority=1) 문단:
  1, 2, 3, 4, 5, 6, 7, 한3.1  ← 적용범위만

DB에 없는 핵심 문단:
  8~17  (정의: 기능통화, 표시통화, 마감환율, 환율 등)
  20~22 (외화거래 최초 인식)
  23~26 (후속 보고기간 말 보고: 화폐성/비화폐성 항목 환산)
  28~34 (외환차이 인식: 당기손익 vs 기타포괄손익)
  38~43 (기능통화→표시통화 환산: 자산/부채 마감환율, 손익 거래일환율)
  44~47 (해외사업장 처분)
  48~57 (순투자, 영업권, 공정가치조정 등)
```

실행 로그에서 Agent가 문단 23·28·39·41을 검색 시도했으나, DB에 존재하지 않아 **5회 검색 (50K 토큰) 후에도 실패**. 결국 사전지식으로 답변.

### 누락 기준서의 공통 패턴

```sql
-- 문제 기준서: main이 적고 transition이 비정상적으로 많음
K-IFRS 1001: main=7,  transition=47   ← 본문이 transition으로 분류?
K-IFRS 1021: main=8,  transition=21
K-IFRS 1028: main=3,  transition=34
K-IFRS 1038: main=8,  transition=37
K-IFRS 1039: main=3,  transition=71   ← transition이 main보다 23배

-- 정상 기준서: transition은 소수이거나 없음
K-IFRS 1037: main=25, transition=0
K-IFRS 1032: main=38, transition=0
```

transition 문단 내용 확인 결과, **실제로 경과규정(58번대~) 문단만 포함**되어 있으며
본문이 transition에 잘못 들어간 것은 아님. **본문 문단 자체가 파싱/적재되지 않은 것.**

## 4. 원인 추정

`_IFRS_parsing` 프로젝트(`/home/shin/Project/_IFRS_parsing/`)의 파싱 파이프라인에서
IAS 계열 기준서 처리 시 문제 발생 추정:

### 가능성 1: docx → 마크다운 변환 실패 (`converter/docx_parser.py`)
- IAS 계열 .docx 파일의 구조가 IFRS 계열과 다름 (CLAUDE.md에 기록됨)
- 본문 섹션 인식이 실패하여 문단 8 이후가 마크다운으로 변환되지 않았을 수 있음

### 가능성 2: 마크다운 → 청크 파싱 실패 (`ingester/md_parser.py`)
- 마크다운은 정상이지만, 섹션 구분(`<!-- component: main -->` 등) 처리에서
  IAS 계열의 다른 편제 방식을 처리하지 못했을 수 있음

### 가능성 3: 적재 시 누락 (`ingester/db_writer.py`)
- 파싱은 정상이지만, DB 적재 과정에서 일부 청크가 누락되었을 수 있음

**확인 방법**:
```bash
# 1. 마크다운 파일에 문단 23이 있는지 확인
grep -n "^23\t\|^23 " output/md/*1021*.md

# 2. 파싱 통계 확인
python ingest.py --parse-only --single "K-IFRS 1021"

# 3. JSON export에서 확인
python -c "
import json
with open('output/search_chunks/KIFRS1021.json') as f:
    data = json.load(f)
paras = [c['para_number'] for c in data['chunks'] if c['component'] == 'main']
print(f'main 문단: {sorted(paras)}')
"
```

## 6. 수정 방안

### 단기 (IFRS_Agent)
- [x] `problems.md`에 기록
- [ ] Agent 프롬프트에 "검색 결과가 적용범위만 반환되면 DB 한계를 안내" 규칙 추가

### 근본 수정 (_IFRS_parsing 프로젝트)
1. 영향받는 8개 기준서의 마크다운 파일 점검
2. `converter/docx_parser.py`에서 IAS 계열 파싱 로직 수정
3. 해당 기준서 재파싱 → 재적재
4. 적재 후 DB 검증 쿼리로 main 문단 수 확인

### 검증 쿼리 (수정 후 실행)
```sql
-- 수정 후 모든 기준서의 main 문단이 최소 10개 이상이어야 함
SELECT standard_id, count(*) AS main_cnt
FROM chunks
WHERE component = 'main' AND authority = 1
GROUP BY standard_id
HAVING count(*) < 10
ORDER BY main_cnt;
-- 결과가 빈 테이블이면 정상
```
